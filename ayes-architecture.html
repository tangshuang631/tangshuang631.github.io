Ayes Architecture

Local monitoring skill architecture for agent-driven desktop observation and memory retrieval

Agent-first interaction

  • • Agent uses lightweight query before raw evidence
  • • Menu bar is auxiliary control and status surface
  • • Task settings remain editable through conversation

Observation pipeline

  • • Capture supports screen, process, window and ROI
  • • OCR and optional VLM converge into MemoryFact
  • • Dedupe and compaction reduce token and disk cost

Local persistence

  • • Runtime data is stored under installed skill directory
  • • Each task owns config, memory, logs and screenshots
  • • SQLite / FTS index supports compact retrieval
