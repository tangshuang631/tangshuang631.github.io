产品概览

产品定位

面向本地使用场景的统一工作入口

核心目标

把常用能力集中在同一界面中 降低切换与理解成本

模块展示

会话工作区

技术栈
React Tauri Ollama 本地模型交互
功能
负责承载用户输入 任务响应 搜索上下文与结果输出
设计思路
把主要交互集中在中间工作区 左侧负责导航 右侧负责状态与任务信息 避免信息散落

搜索与 RAG

技术栈
本地搜索配置 自定义 Provider RAG 上下文接入
功能
支持联网搜索来源配置 搜索测试与来源展示
设计思路
把搜索能力做成可配置入口 而不是默认隐藏功能 让来源 失败回退和后续接入都更清楚

知识库

技术栈
本地文件导入 共享文件库 知识库绑定
功能
支持文件导入 当前知识库管理与后续角色绑定
设计思路
先整理共享文件 再绑定到具体知识库 方便后续给 NPC 和工作区复用

Skills

技术栈
本地技能目录 技能安装 启用与管理
功能
展示推荐技能 已安装技能和后续技能操作入口
设计思路
将技能能力从聊天流程中抽离出来单独管理 让用户知道当前系统具备什么能力

NPC

技术栈
角色配置 知识库绑定 本地模型协作
功能
支持不同角色使用不同知识与能力完成任务
设计思路
把角色定位 知识来源和能力边界分开配置 便于后续扩展不同场景下的工作流

MCP

技术栈
MCP 管理 推荐清单 扩展接入
功能
展示可安装的 MCP 与后续扩展入口
后续方向
继续补充浏览器联动 网页读取和更多工具型扩展 让工作台具备更强的外部操作能力

审计

技术栈
执行日志 审计记录 任务轨迹展示
功能
记录任务执行过程中的关键行为和上下文来源
设计思路
把日志做成可查看的前台页面 而不是只留在底层调试信息中 方便后续排查与回溯

设置

技术栈
Ollama 本地模型配置 全局参数设置
功能
负责本地模型检测 输出预算 模型切换与全局配置入口
设计思路
把系统级能力集中留在设置中 避免和业务页面混杂 同时方便后续扩展更多本地运行参数

当前进展

产品层面

已形成完整的工作台结构和主要页面框架

后续重点

继续完善前端细节 后端链路稳定性和更多本地能力扩展
